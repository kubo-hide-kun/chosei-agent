# ADR 0006: 不正利用対策(合言葉・レートリミット・日次予算)

- ステータス: Accepted
- 日付: 2026-07-11

## 背景

AI 解析エンドポイントは Claude API の従量課金を伴うため、悪意あるユーザーの大量リクエストが
直接コスト(破産リスク)になる。イベント作成もゴミデータ増殖の踏み台になりうる。
身内利用が前提のサービスなので、重い認証基盤は過剰。

## 決定

3 層で守る。実装は `src/server/infrastructure/runtime/security.ts` に集約。

1. **合言葉(アクセスキー)**: 環境変数 `CHOSEI_ACCESS_KEY` を設定すると、
   **イベント作成**と **AI 解析(候補・回答の両方)** に `x-access-key` ヘッダが必須になる(不一致は 401)。
   比較は `timingSafeEqual` による定数時間比較。未設定なら制限なし(ローカル開発モード)。
   キーは UI で入力後 localStorage に記憶する
2. **レートリミット(IP 単位・インメモリ)**: AI 解析 10 回/分、イベント作成 20 回/分、
   出欠回答 30 回/分。超過は 429。バケット総数に上限を設けメモリ枯渇を防ぐ
3. **Claude 日次予算**: `CHOSEI_AGENT_DAILY_LIMIT`(既定 200 回/日)を超えた AI 解析は
   **エラーにせずルールベースへ自動縮退**する。サービスは動き続け、課金だけが止まる

出欠回答の手動送信にはキーを要求しない(URL を受け取った参加者の体験を守る。レートリミットのみ)。

## 理由

- 「身内のみが知る合言葉」はアカウント基盤なしで運用でき、要件(身内限定の実行)に釣り合う最小の仕組み
- 課金の上限は 429 で止めるより縮退の方がよい: 正当なユーザーの体験を壊さず、支出だけを確実に上限で止められる

## 影響・トレードオフ

- レートリミットと予算カウンタはインメモリのため、**単一プロセス前提**。プロセス再起動でリセットされ、
  複数インスタンス構成では台数分に緩む。デプロイ形態が決まったら永続ストア化を別 ADR で判断
- 合言葉は全員で共有する単一キーのためローテーションは手動(env 変更)。漏洩時は変更すれば以降のリクエストを遮断できる
- localStorage 保存のため共用端末では残る。身内利用前提で許容

## 代替案

- **アカウント認証(OAuth 等)**: 身内数人の利用に対して過剰。導入・運用コストが見合わないため不採用
- **AI 解析を回答側だけキー不要にする**: 課金経路が無防備になるため不採用(回答側も Claude を呼ぶ)
- **予算超過時に 429 を返す**: フォールバックで無料に縮退できるのに機能を止めるのは過剰なため不採用

# ADR 0004: CI ゲート(typecheck + テスト + ビルド)

- ステータス: Accepted
- 日付: 2026-07-11

## 背景

手元の確認(typecheck / 絞ったテスト)は速さ優先で範囲を絞る運用のため、
基準線(全テスト + ビルド成功)を機械的に担保する関門が別に必要。

## 決定

GitHub Actions(`.github/workflows/ci.yml`)で、main への push と全 PR に対して次を実行する。

1. `npm run typecheck`
2. `npx vitest run`(全テスト)
3. `npm run build`(本番ビルド)

## 理由

- 手元は速いループ(絞った確認)、CI は基準線(フル確認)という 2 段構えで、開発速度と品質担保を両立する
- リポジトリが小さいうちは全テスト + ビルドでも数分で終わるため、ラベルゲート等の間引きは現時点では不要

## 影響・トレードオフ

- テスト実行時間が伸びてきたら、変更検知による間引きやジョブ分割を別 ADR で判断する
- coverage 閾値は現時点では設けない(テスト追加の規律は AGENTS.md の DoD で担保)。閾値を入れる場合は別 ADR

## 代替案

- **pre-push hook のみ**: ローカル環境差・hook 未設定をすり抜けるため、CI を正本の関門とする
- **ビルドを CI から除外**: Next.js はビルド時にしか出ないエラー(RSC 境界等)があるため含める

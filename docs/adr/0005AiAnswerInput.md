# ADR 0005: 回答側の AI エージェント入力

- ステータス: Accepted
- 日付: 2026-07-11

## 背景

作成側は AI / JSON でまとめて入稿できるのに対し、回答側は候補ごとにボタンを選ぶ操作が残っていた。
候補が多いイベントでは回答側の入力コストも高い。

## 決定

参加者が自然文(例: 「火曜は行けます、金曜は無理、土曜は微妙」)で出欠を書くと、
エージェントが候補ごとの ok / maybe / ng に変換してフォームへ反映する。

- 解析は ADR 0003 と同じ 2 段構え: Claude(gateway: `claudeAnswerGateway.ts`)+ ルールベースフォールバック(domain: `answerText.ts`)
- **解析結果は直接送信しない**。フォームの ◯/△/✕ に反映するだけで、参加者が確認・修正してから送信する(human in the loop)
- 読み取れなかった候補は空のまま残し、推測で埋めない。文中の名前・コメント(遅刻連絡など)も抽出して各欄に補完する
- プロンプトの正本は docs/SYSTEM_PROMPT.md §2。candidates に存在しない ID は application 層で除外する

## 影響・トレードオフ

- AI 解析エンドポイントが 1 つ増え、悪用時のコスト面が拡大する → ADR 0006 の対策(合言葉・レートリミット・日次予算)を同時に適用
- ルールベースは日付・曜日への言及が必須(「行けます」だけでは対象を特定できずエラー)。Claude 利用時はより柔軟

## 代替案

- **解析結果を即送信**: 誤読時に取り消せない(回答修正は未対応)ため不採用。プレビュー反映方式とした

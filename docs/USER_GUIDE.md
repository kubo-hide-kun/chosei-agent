# 使い方ガイド

chosei-agent の利用者向け案内です。イベントの作成から日程決定までの流れを説明します。
(開発者向けの情報は [README.md](../README.md)、入稿 JSON の厳密な仕様は [JSON_FORMAT.md](./JSON_FORMAT.md) を参照)

## 全体の流れ

1. **幹事**: イベントを作成する(AI エージェント or JSON で候補日時をまとめて入稿)
2. **幹事**: 発行された URL を参加者に共有する
3. **参加者**: URL を開き、候補ごとに ◯ / △ / ✕ で出欠を回答する
4. **幹事**: 集計表で最有力候補(★)を確認し、日程を決定する

ログイン・アカウント登録は不要です。

### 合言葉(アクセスキー)について

管理者が合言葉を設定している環境では、**イベント作成と AI 解析(候補の組み立て・回答の読み取り)** に
合言葉の入力が必要です。合言葉はイベントの管理者・グループ内で共有されます。
一度入力するとお使いの端末に記憶されます。出欠の手動回答(◯/△/✕ を選んで送信)には合言葉は不要です。

## 1. イベントを作成する

トップページで 2 つの入稿方法を選べます。

### AI エージェントで入稿(おすすめ)

イベントの内容と候補日程を普通の文章で書くだけで、エージェントが候補日時を組み立てます。

```
チームの暑気払い。来週の火曜と木曜の夜、あと 7/24 の 19時から21時 でお願いします
```

- 「来週の◯曜」「平日」「週末」「今日・明日」などの相対表現は今日の日付を基準に解決されます
- 「夜」は 19:00〜21:00、「昼」は 12:00〜13:00、「朝」は 09:00〜10:00 と解釈されます
- 解析結果は**作成前にプレビューで編集できます**。日付や時間を直してから「この内容でイベントを作成」を押してください
- 日付が読み取れない文章はエラーになります。具体的な日付(7/20 など)や曜日を含めてください

### JSON で入稿

候補が多いとき・スクリプトで生成するときは JSON でまとめて登録できます。

```json
{
  "title": "新年会",
  "description": "会場は渋谷を予定しています",
  "candidates": [
    { "date": "2026-07-21", "start": "19:00", "end": "21:00" },
    "2026-07-24 19:00-21:00",
    "2026-07-25"
  ]
}
```

- `title`(イベント名)と `candidates`(候補日時、最大 100 件)が必須です
- 候補はオブジェクト形式と文字列形式(`"YYYY-MM-DD HH:mm-HH:mm"`)を混在できます
- 形式に誤りがあると、どのフィールドが問題かのエラーが表示されます
- 詳細仕様は [JSON_FORMAT.md](./JSON_FORMAT.md)

## 2. URL を共有する

イベント作成後の画面に共有 URL が表示されます。「URL をコピー」を押して、
チャットやメールで参加者に送ってください。URL を知っている人は誰でも回答できます。

## 3. 出欠を回答する(参加者)

共有された URL を開き、「出欠を回答する」で名前を入力し、候補ごとに選択します。

| 記号 | 意味 |
| --- | --- |
| ◯ | 参加できる |
| △ | 調整すれば参加できる・未確定 |
| ✕ | 参加できない |

コメント欄(任意)には遅刻の連絡や補足を書けます。送信するとすぐ集計表に反映されます。
選択しなかった候補は「回答なし(-)」として記録されます(最低 1 件選択すると送信できます)。

### AI エージェントで回答する

候補が多いときは、自然文で書いて AI に読み取らせることもできます。

```
田中です。火曜は行けます、金曜は無理、土曜は微妙。遅れるかもです
```

- 「AI に読み取らせる」を押すと、読み取った出欠が下の表の ◯/△/✕ に反映されます。名前・コメントも文中にあれば自動で入ります
- **この時点ではまだ送信されていません**。内容を確認・修正してから「回答を送信」を押してください
- 読み取れなかった候補は未選択のまま残ります(推測では埋めません)。手動で選択するか、そのまま送信すると「回答なし」になります
- 日付や曜日(「7/24」「金曜」)を含めて書くと正しく読み取られます。「全部OK」「火曜以外は無理」のような書き方もできます(「火曜以外は無理」の場合、火曜自身は推測せず未選択のまま残します)
- 管理者が合言葉を設定している場合は、合言葉の入力が必要です

## 4. 集計を見て日程を決める(幹事)

「回答状況」の表で、候補ごとの ◯ / △ の数と各参加者の回答が一覧できます。

- **★ が付いた行が現時点の最有力候補**です(◯ = 2 点、△ = 1 点で採点した最高得点の候補)
- 同点の候補が複数ある場合はすべてに ★ が付きます

## よくある質問

- **回答を修正したい** — 現在は同じ名前でもう一度回答すると別の行として追加されます。修正機能は未対応のため、幹事に連絡してください
- **候補を後から追加したい** — イベント編集は未対応です。新しいイベントを作成して URL を共有し直してください
- **AI の解析結果が意図と違う** — 作成前のプレビューで JSON を直接編集できます。日付・時刻を修正してから作成してください
- **「合言葉が正しくありません」と表示される** — イベントの管理者に正しい合言葉を確認してください
- **「リクエストが多すぎます」と表示される** — 短時間に操作しすぎると一時的に制限されます。1 分ほど待ってから再度お試しください

# 入稿 JSON フォーマット

イベント作成 API(`POST /api/events`)・イベント編集 API(`PUT /api/events/{id}`)
およびトップページの「JSON で入稿」タブ、イベント画面の「イベント内容を編集」で使用する形式です。
作成・編集で body の形は同一です。

## 全体構造

```json
{
  "title": "新年会",
  "description": "会場は渋谷を予定しています",
  "candidates": [
    { "date": "2026-07-21", "start": "19:00", "end": "21:00" },
    { "date": "2026-07-22", "start": "19:00", "end": "21:00", "label": "第2候補" },
    "2026-07-24 19:00-21:00",
    "2026-07-25"
  ]
}
```

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `title` | string (1〜200文字) | ✓ | イベント名 |
| `description` | string (〜2000文字) | - | 補足説明 |
| `candidates` | array (1〜100件) | ✓ | 候補日時。オブジェクト形式と文字列形式を混在可 |

## 候補日時: オブジェクト形式

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `date` | `YYYY-MM-DD` | ✓ | 候補日。**カレンダー上に実在する日付のみ**(`2026-02-31` などは 422) |
| `start` | `HH:mm` | - | 開始時刻 |
| `end` | `HH:mm` | - | 終了時刻。**`start` より後であること**(深夜跨ぎは未対応) |
| `label` | string (〜100文字) | - | 表示ラベル。省略時は `7/21(火) 19:00〜21:00` の形式で自動生成 |

## 候補日時: 文字列ショートハンド

| 形式 | 例 |
| --- | --- |
| `YYYY-MM-DD` | `"2026-07-25"` |
| `YYYY-MM-DD HH:mm` | `"2026-07-25 19:00"` |
| `YYYY-MM-DD HH:mm-HH:mm` | `"2026-07-25 19:00-21:00"` |

## API レスポンス

成功時 — 作成 (201) / 編集 (200):

```json
{ "id": "Ab3xYz012345", "url": "/events/Ab3xYz012345" }
```

バリデーションエラー時 (422):

```json
{
  "error": "入稿 JSON がスキーマに一致しません",
  "issues": [{ "path": "candidates.0.date", "message": "date は YYYY-MM-DD 形式で指定してください" }]
}
```

## イベント編集(`PUT /api/events/{id}`)

title・description・candidates を丸ごと置き換える。存在しない `id` は 404。
アクセス制御はイベント閲覧・回答と同じで、URL(`id`)を知っていれば誰でも編集できる
(合言葉は不要。詳細は [ADR 0011](./adr/0011EventEditing.md))。

候補日時は `(date, start, end)` の組み合わせで既存候補と照合し、
一致するものは同じ候補として扱われ、その候補への既存の回答は保持される。
一致しない(削除された)候補への回答は破棄される。
